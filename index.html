Cours GIT

Qu'est ce que GIT ?

GIT est un outil de versionning, et permet d'avoir une traçabilité et un historique de tous les fichiers du projet.

Il permet également de travailler en équipe: envoyer nos modifications à nos collegues et récuperer nos modifications des collegues.

Dans la traçabilité des fichiers, GIT nous donne toujours 3 info: Qui ? Quand? Pourquoi ?

Les commandes de base

Initialiser un projet:
git init
Voir le statut des fichiers
git status
Ajouter des fichiers
git add 'fichier'
Ajouter TOUS LES FICHIERS :
git add .
Commit (enregistrer) les fichiers ajoutés
git commit
Un commit necessite toujours un message
commit avec message:
git commit -m "message'
commit avec add + message (fonctionne sur les fichers déjà add précedemment):
git commit -a -m "message"

Correction des fichiers

Voir les differents commit:
git log
Corriger les modifications sur un fichier (pas encor add) :
git checkout 'fichier'
Supprime un add :
git reset 'fichier'
Supprime tous les add :
git reset .
Supprimer le dernier comit :
git reset HEAD^
Supprimer les deux derniers commit :
git reset HEAD^

La notion de branche

une branche est une copie du projet, elle permet de developper en parallele de nouvelle fonctionnalités. Lorsqu'on considère les nouveautés opérationnelle, on peut les rappatrier sur la version principale (master)

Voir les branches:
git branch
Créer une nouvelle branche :
git branch 'nom'
Se placer sur une branche :
git checkout 'nom de la branche en question'
Voir les différences entre deux branches:
git diff 'nom'..master
Apporter les modifications effectees sur une branche dans master (il faut etre sur master):
gift checkout master
git merge 'nom'
Remplacer master par une branche:
git checkout master
git fetch. 'nom'
le fetch va literallement remplacer master par la branche. Il s'agit donc d'une grosse modification dans le projet (ex: version 2)
